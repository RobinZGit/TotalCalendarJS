## 1. Подстановка работает только через `[eval]…[eval]`

`calcEval` обрабатывает онлайн-вещи **только** там, где в строке есть пары `[eval]` — между ними вызывается `eval(...)`. Если в тексте просто написано `_прошло_секунд_` или `_пульс_` **без** обёртки `[eval]…[eval]`, это для движка **не выражение**, а обычные символы: они **нигде не подставляются числом**, в массиве так и остаётся буквальная подстрока (и в `say()` тоже).

То есть модель «имя само по себе — это переменная внутри строки» в JS так не устроена: переменные не подставляются в строку сами, нужен либо `[eval]`, либо явная склейка/шаблон.

## 2. «Только во время say()» сейчас не выполняется: второй проход уже «запекает» значения

В `finish_Gl_aRithmLisp` после сборки массива идёт блок «ВТОРОЙ ПРОХОД»: по всему `Gl_aRithmLisp` в цикле вызываются `calcEval("N", …)` и `calcEval("S", …)` для **каждой** пары интервал/текст, при этом обновляются `_прошло_*`, `_номер_итерации_`, вызывается `syncOnlinePulseParam()` и т.д.

То есть всё, что попало в `[eval]` и зависит от `_прошло_секунд_` / `_пульс_` / итерации, **один раз считается при подготовке массива** (по сути «прогон сценария» в момент генерации), и **результат записывается прямо в элементы `Gl_aRithmLisp`**.

Во время реальной тренировки пульс может меняться, но текст уже не содержит живого `[eval]` с `_пульс_`.

## 4. Дополнительно: `_номер_итерации_` и два места инкремента

`_номер_итерации_` увеличивается и во втором проходе в `finish_Gl_aRithmLisp`, и при срабатывании таймера в `fSayInTime`. ## 1. После сборки в GitHub Actions (самый типичный сценарий)

После шага `Upload APK artifact` можно добавить шаг с SMTP, например [dawidd6/action-send-mail](https://github.com/dawidd6/action-send-mail):

- В **Secrets** репозитория: `MAIL_SERVER`, `MAIL_USERNAME`, `MAIL_PASSWORD`, при необходимости порт и TLS.
- В workflow: `to: your@email.com`, `attachments: TotalCalendar.apk`.

**Ограничения:**

| Проблема | Что учесть |
|----------|------------|
| Размер APK | У Gmail ~25 МБ на письмо; у многих SMTP лимит 10–25 МБ — крупный APK может не уйти |
| Секреты | Пароль приложения / API-ключ только в Secrets, не в коде |
| Спам | Письма с `.apk` часто режутся фильтрами |

**Надёжнее для больших файлов:** artifact + ссылка на скачивание (GitHub Releases, облако) и в письме только ссылка, а не вложение.

## 2. Локально после `build_apk.ps1`

Тот же SMTP (Send-Mail в PowerShell, `sendmail`, Python + `smtplib`) — логика та же: нужен сервер и учётные данные на вашей машине.

## 3. Из самого Android-приложения

Технически можно открыть системный «Поделиться» / `Intent.ACTION_SEND` с APK, но:

- пользователь сам выбирает почтовый клиент и адрес;
- для автоматической отправки на **фиксированный** адрес из приложения нужны свои сервер/SMTP и обычно это неудобно и небезопасно.

4. **Android** — в `AndroidManifest` расширен список `configChanges`, чтобы смена размера/ориентации реже пересоздавала activity.

- Если превью в IDE **полностью перезагружает** iframe (это поведение редактора), восстановление сработает **после** перезагрузки — тренировка должна вернуться сама.
